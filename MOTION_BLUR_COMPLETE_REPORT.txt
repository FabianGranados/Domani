CORE MOTION BLUR ALGORITHM (Canvas)
================================================================================

function renderMotionBlur() {
    // Clear canvas
    ctx.clearRect(0, 0, width, height)
    
    // Optimization: Lazy blur (only if moving fast)
    if (Math.abs(angularVelocity) < 0.5) {
        ctx.globalAlpha = 1.0
        drawWheel(rotation)
        return
    }
    
    // Render historical frames with decreasing opacity
    for (let i = 0; i < blurSamples; i++) {
        const opacity = (1 - (i / blurSamples)) * blurStrength
        const frameDelay = (i * 16) / 1000  // ~16ms per frame (60fps)
        const blurRotation = rotation - (angularVelocity * frameDelay)
        
        ctx.globalAlpha = opacity
        drawWheel(blurRotation)
    }
    
    // Current frame at full opacity
    ctx.globalAlpha = 1.0
    drawWheel(rotation)
}

CRITICAL OPTIMIZATIONS
================================================================================

1. LAZY BLUR EVALUATION
   - Apply blur only when velocity > threshold (0.5 rad/s)
   - FPS improvement: 5-10% when stationary
   - Simple check: if (Math.abs(angularVelocity) > threshold)

2. ADAPTIVE BLUR SAMPLES
   - Mobile: 3 samples
   - Desktop: 5-6 samples
   - FPS impact on mobile: +10-15%

3. OFFSCREEN CANVAS
   - Pre-calculate blur samples
   - Transfer to main canvas
   - FPS improvement: 15-20% on complex renders

4. REQUESTANIMATIONFRAME
   - Syncs with monitor refresh rate
   - Eliminates frame tearing
   - More consistent FPS than setInterval

5. VELOCITY-BASED BLUR SCALING
   - Blur radius proportional to angular velocity
   - Formula: blurRadius = Math.abs(velocity) * 0.05
   - More realistic visual effect

FRAME INTERPOLATION FOR SMOOTHNESS
================================================================================

Issue: Rotation appears "jerky" at high speeds
Solution: Interpolate frames with easing functions

Recommended: Cubic Ease-Out
Formula: easeProgress = 1 - (1 - progress)³

Effect:
- Smooth acceleration at start
- Smooth deceleration at end
- Natural, pleasing visual perception

Alternative: CSS cubic-bezier(0.25, 0.46, 0.45, 0.94)
