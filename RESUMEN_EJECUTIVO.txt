3. EASING CURVES - FORMULAS MATEMATICAS

================================================================================
FORMULAS CLAVE VALIDADAS
================================================================================

A. EXPONENTIAL DECAY (General)
   N(t) = N₀ × e^(-λt)
   - N₀ = Velocidad inicial
   - λ = Constante de decaimiento
   - t = Tiempo

B. ITERATIVE DECAY (Frame-based)
   v(n+1) = v(n) × decay_factor
   - Más eficiente para games
   - decay_factor ∈ [0.92, 0.99]

C. DECELERATION FORMULA (Ruedas)
   a = -v² / (2d)
   - a = Desaceleración (rad/s²)
   - v = Velocidad inicial (rad/s)
   - d = Distancia de parada (rad)

D. FRAMES TO STOP CALCULATION
   n = log(threshold) / log(decay_factor)
   - A 60 FPS: decay 0.96 = ~102 frames = 1.7 segundos

E. CUBIC BEZIER (Animation Curves)
   cubic-bezier(x1, y1, x2, y2)
   - Rango: x ∈ [0,1], y puede ser > 1
   - Puntos fijos: (0,0) start, (1,1) end

PARAMETROS TECNICOS POR ENGINE

UNITY:
- DOTween duración: 0.5-2.0 segundos típico
- Ease.OutQuad para UI / Ease.OutCubic para juegos
- Decay factor equivalente: ~0.96-0.98

PHASER 3:
- 32 easing functions built-in
- Sine.easeOut o Quad.easeOut recomendado
- Duración típica: 2-4 segundos para ruedas

UNREAL ENGINE:
- Angular Damping: 0.5-5.0 rango práctico
- Linear Damping: 0.5-1.0 para movimiento
- Physics bodies heredan estos valores

GAMEMAKER:
- Decay iterativo: angular_velocity *= 0.96
- Image angle update: image_angle += angular_velocity
- Easing curves disponibles via curves built-in

FLUTTER:
- Duration: 3-5 segundos para ruedas
- CurvedAnimation + AnimationController
- Curves.easeOutCubic o Curves.easeOut recomendado

================================================================================
RUEDAS GIRATORIAS - BEST PRACTICES
================================================================================

CONFIGURACION ESTANDAR:
- Duration: 3-5 segundos
- Decay Fase 1 (50% inicial): 0.94
- Decay Fase 2 (50% final): 0.97
- Rotaciones: 8-12 revoluciones completas
- Easing: EasingSinOut o EaseOutQuad
- Threshold parada: 0.001 rad/seg

IMPLEMENTACION ITERATIVA (Recomendado):
1. velocity *= decay_factor cada frame
2. angle += velocity
3. Detener si |velocity| < threshold
4. Más eficiente que tweens complejos
5. Control fino sobre "feel" de animación

================================================================================
CONCLUSIONES
================================================================================

1. DECAY FACTOR 0.96 es el equilibrio estándar en games
   → 4% fricción por frame
   → ~1.7 segundos a parada total @ 60 FPS

2. EASING FUNCTIONS varían según contexto
   → Quadratic para UI / Cubic para gameplay
   → Sine para suavidad / Exponential para impacto

3. RUEDAS GIRATORIAS usan two-phase deceleration
   → Fase rápida inicial + fase lenta final
   → Crea sensación más natural

4. RESTITUTION vs FRICTION son independientes
   → Restitution = elasticidad en colisión (0-1)
   → Friction = resistencia superficial (0-1)

5. ANIMACIONES iterativas > Tweens complejos
   → Mejor control
   → Mejor performance
   → Más predecible visualmente
